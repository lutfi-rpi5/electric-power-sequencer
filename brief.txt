Nama Project : Electrical Sequencer System

Overview Alat
(Functions & Features): 1. General Functions
                        - Alat ini berfungsi untuk menyalakan 8 Output Outlet 220VAC secara berurutan/sequencing dari 8 Channel relay yang dikontrol oleh ESP32 Devkit V1
                        - Alat ini juga berfungsi sebagai proteksi: Under/Overvoltage (hanya Master Source) dan Overcurrent (per-channel Output Outlet) yang bisa disetting threshold nya di Menu Config
                        - Alat ini dikontrol penuh secara lokal tanpa IoT dengan menggunakan Interface LCD I2C 20x4 dan 4 Push Button untuk kontrol Menu di LCD
                        - Alat ini mengedepankan user-friendly untuk target user menengah-awam
                        
                        2. Features
                        - dilengkapi dengan 2 Level Config: Programmer Level Config - config.h & User Level Config - Menu Config lewat LCD
                        - memiliki Display Monitoring dan Menu Config 
                        - memiliki 4 Push Button (NO, not latching), 1 tombol khusus untuk ON/OFF Output Outlet dan 3 untuk Navigasi Menu (<, Enter/Back, >)
                        - dilengkapi dengan 1 buah PZEM-004T-100A (UART) untuk Master Source dengan (parameter yang dipakai: Voltage, Current, Frequency, Active Energy) dan 8 buah sensor Arus ACS712 untuk Arus per channel Output Outlet AC

Spesifikasi Alat
(Electrical Hardware) : 1. Controller
                        - ESP32 Devkit V1 30 Pin
                        
                        2. Input
                        - 1x PZEM-004T 100A (UART) (Master Source) 
                          = PZEM adalah module paling depan dari semua sensor dan berfungsi untuk mengecek tegangan dan arus sebelum tegangan dan arus tersebut di distribusikan ke 8 channel outlet
                        - 8x ACS712 Analog Current Sensor (Outlet Current Sensor)
                          = ACS712 adalah module sensor arus untuk mengecek arus AC setiap channel dan memastikan arus AC setiap channel tetap di bawah threshold config masing-masing channel (setiap channel memiliki threshold yang berbeda-beda, ada yang CH1=10A, CH2=15A, CH3=5A, dst, tergantung user config)
                        - 2x ADS1115 I2C (masing-masing ADS1115 handle 4 sensor ACS712)
                          = ADS1115 digunakan sebagai ADC external untuk menerima input analog dari 8 channel ACS712 untuk mendapatkan ke presisian dan resolusi data yang jauh lebar dan lebih baik dari Built-in ADC ESP32, setiap 1 buah ADS1115 menghandle 4 buah sensor ACS712 dan dibedakan setiap address I2C ADS1115 
                        - 4x Push Button Normally Open, Not Latching (1 button untuk ON/OFF Sequence_Button, 3 button untuk Menu_Button)
                        
                        3. Output
                        - 8 Channel Active Low 5V Optocoupler Relay (Always OFF saat booting)
                          = relay ini berfungsi untuk bridging/ menyalakan relay yang ampere nya lebih besar untuk arus AC, maka dari itu buat di Programmer Config

                        4. Display
                        - 1x LCD 20x4 I2C

Spesifikasi Sistem yang akan di desain/program:
1. Semua bagian sistem dan fungsi harus FULL asynchronous/non blocking system tanpa delay()
2. [hardcoded & menu config] sentralisasi semua config dalam 1 file config.h dan pisahkan section config yang hardcoded (hanya bisa diakses oleh programmer lewat program) dan config yang bisa diakses/modifikasi lewat lcd menu yang akan tersimpan di EEPROM esp32
3. [Push Button] Push Button pada alat ini ada 4:
  - Sequence Button/SQC_BTN = hanya memiliki 1 fungsi khusus, yaitu untuk menghidupkan/mematikan 8 Channel Outlet secara sequence
                              detail fungsi:  - bukan tombol untuk navigasi menu
                                              - tombol ini bisa menghidupkan/mematikan 8 channel outlet di mana saja, bisa saat di "Monitoring Mode" maupun saat "User Config Mode"
                                               - saat tombol ini ditekan dengan waktu selama 2 detik (bisa di custom di user config), maka:
                                                 a. PZEM akan cek tegangan master source apakah dalam range normal (antara UV dan OV)
                                                  b. Jika tegangan dalam range normal dan channel masih OFF semua → mulai sequence ON (nyalakan channel berurutan sesuai interval, channel dengan limit 0.0A akan di-skip)
                                                  c. Jika tegangan di luar range (UV/OV) → tampilkan alert di LCD, jangan mulai sequence
                                                  d. Jika channel sudah ON semua → tekan SQC_BTN lagi 2 detik → sequence OFF (matikan channel berurutan sesuai interval, channel dengan limit 0.0A sudah pasti OFF dan tetap di-skip)
                                                  e. Jika sedang dalam proses sequence (ON/OFF berlangsung) → tekan SQC_BTN diabaikan (debounce)

  - Left Button             = berfungsi untuk navigasi menu/kursor menu/config selector
  - Center Button           = berfungsi untuk confirm(OK) button/enter button/back button, jika long press selama 1 detik (config.h) maka akan menjadi back button, jika short press akan menjadi enter button
  - Right Button            = berfungsi untuk navigasi menu/kursor menu/config selector

3c. Config Mode: Scene 3 (Set Limit Arus per channel, presisi 1 desimal)
 *   ┌────────────────────┐
 *   │CH1 6.0    CH2 1.0  │  ← 
 *   │CH3 1.0    CH4 0.0  │  ←
 *   │CH5 0.0    CH6 0.0  │  ←
 *   │CH7 0.0    CH8 0.0  │  ←
 *   └────────────────────┘
     Aturan pembagian limit arus per channel:
     - Setiap channel bisa diisi nilai 0.0A - max tersisa
     - TOTAL seluruh channel TIDAK BOLEH melebihi I Limit PZEM
     - Channel dengan nilai 0.0A = channel tidak mendapat alokasi beban
       → channel tsb akan dilewati (skip) saat SQC sequence berjalan
     - Contoh: I Limit PZEM = 8A, setting CH1=6A, CH2=1A, CH3=1A (total=8A)
       → CH4-CH8 = 0.0A (tidak mendapat jatah, otomatis di-skip sequence)
     - Kombinasi bebas: tidak harus channel 1-3 saja, bisa CH1=2A, CH5=4A, CH8=2A (total=8A)
       → sequence hanya akan menjalankan channel yang memiliki nilai > 0.0A
     - Setting ini bisa dimodifikasi kapan saja, bahkan saat sequence/load sedang ON

5. [Protection System] Proteksi pada alat ini terdiri dari 3 level:
   5a. Master Source Protection (PZEM-004T):
       - Overvoltage (OV): saat tegangan PZEM ≥ threshold OV
       - Undervoltage (UV): saat tegangan PZEM ≤ threshold UV
       - Overcurrent (I Limit): saat arus PZEM ≥ threshold I Limit
       - Action untuk OV/UV/ILimit: bisa diatur di config.h (hardcoded):
         a) TRIP_ALL + Alert: seluruh channel TRIP dan tampilkan "!" di Scene 2a
         b) Alert Only: tampilkan "!" di Scene 2a, channel tetap berjalan
       - Selama threshold terlanggar, "!" di Scene 2a akan tampil
       - Saat kondisi kembali normal, TRIP_ALL bisa di-reset di Scene 3 (sama seperti reset trip biasa)

   5b. Per-channel Overcurrent Protection (ACS712 + ADS1115):
       - Setiap channel memiliki threshold arus masing-masing (diatur di Set Limit Arus CH)
       - Jika arus channel ≥ threshold → channel auto TRIP (hanya channel itu, tidak affect channel lain)
       - Status channel berubah menjadi TRIP, ditampilkan di Scene 3

   5c. Trip Recovery (via Scene 3):
       - Untuk channel yang sedang TRIP:
         a. masuk ke write mode di Scene 3 (tekan CENTER short press)
         b. pilih channel yang TRIP, tekan CENTER
         c. pilih OFF dulu, tekan CENTER (matikan beban)
         d. lalu pilih ON, tekan CENTER (nyalakan ulang)
         e. jika channel normal kembali → status ON, kedipan berhenti
       - Untuk channel ON yang ingin dimatikan manual, lakukan langkah yang sama
       - Long press CENTER untuk kembali ke read-only mode

6. [NVS Config Storage] Semua user config disimpan di Non-Volatile Storage (NVS) ESP32 via Preferences library dengan struktur:
   - namespace: "config"
   - key-value pairs per field:
       "ov_thresh"       → float  (Overvoltage threshold V)
       "uv_thresh"       → float  (Undervoltage threshold V)
       "i_limit"         → float  (PZEM Current Limit A)
       "ch_lim_0..7"     → float  (Per-channel current limit A)
       "sqc_int_0..7"    → uint16_t (Sequence interval per channel ms)
       "auto_cal"        → bool   (Auto calibration ON/OFF)
       "sqc_btn_dly"     → uint16_t (SQC button hold time ms)
       "blink_int"       → uint16_t (Trip blink interval ms)
       "op_scene2"       → bool   (Opening scene 2 ON/OFF)
   - Atau bisa juga dengan putBytes/getBytes untuk struct utuh:
       struct UserConfig {
         float ov_threshold;
         float uv_threshold;
         float i_limit;
         float ch_current_limit[8];
         uint16_t sqc_interval[8];
         bool auto_cal_enable;
         uint16_t sqc_btn_delay;
         uint16_t blink_interval;
         bool opening_scene_2_enable;
       };
   - Pada first boot (NVS kosong), default values dimuat dari config.h
   - Validation: gunakan CRC/checksum untuk deteksi corruption

7. [State Machine] Sistem berjalan dengan state machine asynchronous:
   - STATE_IDLE          → saat boot, inisialisasi hardware
   - STATE_OPENING       → menampilkan opening scene 1 & 2
   - STATE_CALIBRATION   → auto-zero calibration (jika enabled)
   - STATE_MONITORING    → main loop: display scene 2a/2b/2c (endless scroll LEFT/RIGHT)
   - STATE_CONFIG        → menu config (enter dari monitoring, long press CENTER back)
   Setiap state non-blocking dengan millis() timer, tanpa delay()

8. [Self-Healing & Memory Optimization] Untuk memastikan uptime ESP32 sangat lama (berhari-hari hingga bulanan) tanpa restart:
   8a. Memory Management:
       - DILARANG menggunakan String class, malloc/free dinamis, new/delete
       - Semua buffer harus static/fixed-size (char array, bukan String)
       - Hindari sprintf, gunakan snprintf dengan buffer fixed
       - Semua variable global/LCD string dialokasikan static di awal

   8b. Heap Monitoring:
       - heap_caps_get_free_size(MALLOC_CAP_INTERNAL) dipanggil periodik (misal tiap 10 menit)
       - Jika free heap turun di bawah threshold (misal < 20% dari total awal) → restart otomatis
       - Log free heap ke Serial untuk debugging (dapat di-enable/disable di config.h)

   8c. Task Watchdog:
       - Aktifkan ESP32 Task Watchdog Timer (TWDT) untuk mendeteksi hang
       - Setiap state machine loop harus feed watchdog (timerAlarmWrite)
       - Jika ada task yang blocking > threshold, WDT akan auto-reset ESP32

   8d. I2C Bus Recovery:
       - Jika komunikasi I2C (LCD/ADS1115) gagal → reset bus I2C, coba reconnect ulang
       - Setelah 3x gagal berturut-turut → tampilkan error screen, jangan crash

   8e. UART/PZEM Recovery:
       - Jika PZEM tidak merespon → restart UART, coba inisialisasi ulang
       - Setelah 3x gagal → tampilkan error "PZEM Error!" di LCD, system tetap jalan

   8f. Auto-Restart Terjadwal (optional):
       - Config.h: opsi enable/disable auto-restart setiap N jam (default: disabled)
       - Jika enabled, restart dilakukan saat channel dalam kondisi OFF semua
